### Bugs Found and Fixed
1. **Independent picks had no-duplicate fallback pulling from coordinated candidates** — fixed. Independent path now uses pWinNow-ranked candidates from `allocation[]`, not composite score.
2. **Dual elimination tracking missing** — fixed. Coordinated and independent paths now maintain separate alive-entry sets. Each path tracks its own eliminations based on its own pick outcomes.
3. **Independent Score column showing "—" in modal** — was rendering `overallScore100` (composite score) instead of `pWinNow`. Claude Code fix dispatched (pending push/deploy).

### Fundamental Algorithm Problem Identified
The coordinated algorithm is **not** chalk-aware. Both coordinated and independent paths always pick from the same top-5 teams by win probability. The only difference is ordering across entries. This means:
- Same 5 teams = same game outcomes = near-identical survival probabilities
- Coordinated path is sometimes WORSE than independent because it assigns lower-pWin teams to some entries for diversification without any chalk-avoidance benefit
- `GAMMA` (chalk-correlation penalty weight) is currently `0.0` — disabled in `portfolioRecommendation.ts`

### Root Cause
The coordinated algorithm diversifies entries against *each other* but not against *the field*. The real value of coordination is: when chalk teams (high field ownership) lose, coordinated entries that avoided them survive while the field gets decimated. Without GAMMA enabled, this doesn't happen.

### What needs to happen next
1. **UI fix (in progress):** Independent Score column → show pWinNow as percentage
2. **Algorithm fix (pending Mike's go-ahead):** Enable and tune GAMMA in `server/services/portfolioRecommendation.ts` so the coordinated path penalizes high-ownership chalk teams, causing genuine team-level divergence from the independent path
3. **UI fix:** Show independent picks for weeks after coordinated path is eliminated

### Success criteria (Mike's definition)
6%+ improvement in overall survival probability across all entries for a given pool = roughly 1 additional week of survival. This is the bar for "meaningful benefit."
